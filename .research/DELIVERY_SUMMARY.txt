RECOMMENDATIONS (Source 4: Best Practices)
--------------------------------------------------------------------------------

1. Official Rust Test Organization (Rust Book Ch. 11)
   → Shared utilities in separate crate (probe has 24 crates)
   → Validated: tokio-test, serde_test, tracing pattern

2. Builder Pattern for Tests (Rust Patterns Book)
   → Tests need ergonomics, not compile-time safety
   → Provide preset methods with sensible defaults

3. Common Test Utilities Pattern (Rust API Guidelines)
   → Use clear naming: event() not make_test_event()
   → Document test utilities

4. Property-Based Testing Patterns (Luca Palmieri)
   → Define strategies once, reuse across tests
   → Share in test utils, not per-test

5. Test Organization Anti-Patterns (Jon Gjengset)
   → Anti-pattern: Copy-pasting test setup (probe has this)
   → Best practice: Separate test utils crate

6. Test Utilities Crate Pattern (tokio, serde, tracing)
   → Expose builders, fixtures, assertion helpers
   → Used as [dev-dependencies]
